# Branding Guidelines

## Core Identity
**Name:** Nazrul Islam
**Titles:** Competitive Programmer, Full Stack Developer, Open Source Contributor, .NET Backend Engineer.
**Primary Vibe:** Professional, algorithmic, sleek, dark-mode focused.

## Color Palette (GitHub Native)
- **Background (Dark):** #0D1117
- **Background (Light):** #ffffff
- **Primary Accent (Blue):** #58a6ff (Dark) / #0969da (Light)
- **Secondary Accent (Purple):** #bc8cff (Dark) / #8250df (Light)
- **Text Primary:** #ffffff (Dark) / #24292f (Light)
- **Text Secondary:** #8b949e (Dark) / #57606a (Light)

## Typography
- **Headings/Display:** Inter or Segoe UI (Sans-serif, clean, modern)
- **Code/Tech:** Fira Code (Monospace, algorithmic feel)

## Visual Elements
- **Banners:** Use a subtle grid pattern to represent structure and algorithms, with abstract glowing elements.
- **Corners:** Rounded corners (15px to 8px) to match modern GitHub UI.

# Asset Naming Conventions
All files in the ssets/ directory must follow this strict naming schema:

### General Format
<category>-<identifier>[-<theme>].<extension>

### Categories
- anner: Hero banners or wide graphics.
- icon: Custom SVGs for skills or UI elements.
- logo: Personal branding logos.
- shot: Project screenshots.

### Themes (Required for theme-aware assets)
- -dark
- -light

### Examples
- anner-hero-dark.svg
- anner-hero-light.svg
- icon-codeforces.svg
- logo-primary.png
- shot-shopfinity.webp

> **Note:** Prefer .svg for vector graphics and .webp for raster images (screenshots) to ensure fast load times.
